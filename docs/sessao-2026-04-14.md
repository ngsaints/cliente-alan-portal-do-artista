## Problema de Login do Admin
### Diagnóstico
- Credenciais corretas: usuário `admin`, senha `1234`.
- API funcionando corretamente (testado via curl).
- Problema identificado: O frontend não estava enviando cookies de sessão nas requisições fetch.
- Falta de `credentials: 'include'` nas chamadas de API.

### Correção Aplicada
- Editado `artifacts/alan-ribeiro-catalog/src/pages/Admin.tsx`:
  - Adicionado `request: { credentials: 'include' }` aos hooks `useLogin`, `useLogout`, `useGetAuthStatus`, `useListSongs`, `useCreateSong`, `useDeleteSong`.
- Reconstruído o frontend com `pnpm --filter @workspace/alan-ribeiro-catalog run build`.
- Implantado via cópia para `/var/www/portal-do-artista` e reload do nginx.

### Problemas Técnicos Encontrados e Resolvidos
- Erros de TypeScript em `lib/api-zod`: Adicionado `"dom"` à lista de libs em `tsconfig.base.json` para suportar tipos `File` e `Blob`.
- Configuração do Vite requer variáveis de ambiente: Usado `PORT=3000 BASE_PATH=/` para build.
